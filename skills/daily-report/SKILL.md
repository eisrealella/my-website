# Skill: daily-report

自动化生成海粼酱的工作日报，写入飞书文档。

## 触发条件
- 用户请求写日报
- cron 定时任务触发

## 两个时间的意义
- **早上 10:00**：回顾前一天一整天的内容，补写前一天晚上的任务
- **晚上 18:00**：总结今天一整天所做的

**重要**：永远保证日报不重复。同一天只写一篇日报，不要因为早晚会触发两次就写两篇。

## 输入
- `doc_token`: 飞书文档 ID（默认：XdQSdchsUoPK5hxveSKc6XEcn6c）
- `date`: 日期（默认：今天，格式 YYYY年M月D日）
- `time_of_day`: `morning` | `evening`（自动从时间判断）

## 输出
写入飞书文档的内容块

## 模板

```markdown
## 📅 {日期}

💭 [一句话感悟，没有前缀，加你喜欢的 emoji]

### 🟢 今日完成

**核心任务**
- [核心工作1描述]
- [核心工作2描述]
- [核心工作3描述]

**技术细节**
- [配置/代码变更具体值]
- [技术决策点]
- [工具/技能使用情况]

### 🔵 明日计划

- [ ] [计划任务1]
- [ ] [计划任务2]

### 💡 思考与复盘

**做得好的**
- [成功点1]
- [成功点2]

**可以更好的**
- [改进点1]
- [改进点2]
```

## 执行步骤

1. **判断时间**：
   - 如果是 10:00（早上）→ 写**前一天**的日报
   - 如果是 18:00（晚上）→ 写**今天**的日报
2. **检查是否已存在**：先读取文档，确认当天日报是否已存在，避免重复
3. **生成内容**：按模板生成，日期标题只用 `📅 YYYY年M月D日`
4. **高亮块**：
   - 使用飞书 callout 块（block_type=19）
   - emoji 选择根据感悟内容风格（如 🫠 🤯 💡 🌊 🐱）
   - 只放一句话感悟，**不要加标题或前缀**
5. **写入**：使用 `feishu_doc append` 写入
6. **验证**：确认写入成功

## 注意事项
- 📅 标题只用日期，**不要加"工作日报"后缀**
- 💭 高亮块放在日期标题**正下方**，无前缀，只有一句话 + emoji
- 核心任务用 bullet 点列表
- 技术细节记录具体配置值、命令、参数
- 明日计划用 checkbox 格式 `□`
- 思考与复盘必填，体现 AI 自主思考
- 按日期逆序插入（新的在前）
- **同一内容不重复写入**

## 示例输出

```
## 📅 2026年2月18日

💭 配置问题就像幽灵，要么不出现，要么出现在意想不到的地方 🫠

### 🟢 今日完成

**核心任务**
- OpenClaw 配置调整：禁用 flawed-avatar 插件
- 分块流式传输上线：feishu 渠道启用 blockStreaming
- 日报系统修复：定位 cron 失败根因并测试验证

**技术细节**
- 模型：`minimax/MiniMax-M2.1` → `minimax-custom/MiniMax-M2.5`
- 投递：添加用户 ID 到 delivery 配置
- ackReactionScope 确认为 `all`

### 🔵 明日计划

- [ ] 观察定时任务执行
- [ ] 优化日报质量

### 💡 思考与复盘

**做得好的**
- 快速定位两层失败原因
- 配置 patch 精确更新

**可以更好的**
- 增加技术细节记录
```

## 依赖工具
- `feishu_doc`: read, append, list_blocks
- `exec`: 日期计算（如需要）
